Flask-Firewall

A web firewall that is configured to protect Flask web applications from common vulnerabilities such as DDoS attacks(Denial of service), SQL and XSS injections, brute force attacks, and CRSF attacks. It enforces rate limits and failed login attempt restrictions to prevent DDoS and brute force attacks. If these limits are surpassed, the firewall sends a warning to the violating IP address and temporarily blacklists it. Repeated violations result in the IP address being permanently blacklisted. Additionally, the firewall sanitizes all user input to detect and block malicious payloads or code injection attempts. If a payload attack is detected, the IP address is permanently blacklisted to prevent future attacks. Moreover, the firewall generates and validates CSRF tokens to simplify the inclusion of CSRF protections.
